05b — Modelagem Supervisionada (Independente)¶

Objetivo: treinar modelos supervisionados sem leakage por construção. Este notebook implementa a Opção B discutida na análise crítica do projeto:

  1. Label honesta derivada de regras de negócio (não do Isolation Forest)
  2. Features independentes das variáveis usadas na composição da label
  3. Comparação entre 3 algoritmos: KNN, Árvore de Decisão e Random Forest
  4. Comparação adicional com o Isolation Forest aplicado aos mesmos dados

Por que este notebook existe: o modelo treinado em 05_modelagem_supervisionada.ipynb atingiu métricas próximas de 1.0 (ROC-AUC=0.9996, F1=0.94), o que revelou um problema de circularidade: o supervisionado aprendia o rótulo gerado pelo próprio Isolation Forest, usando as mesmas features. Este notebook corrige esse problema.

2. Definição da label honesta¶

A label y_honesta é definida pela disjunção de dois critérios de negócio:

  • Critério 3 (fracionamento): valor entre R$ 17.500 e R$ 17.700, próximo ao limite legal de dispensa por valor (R$ 17.600 segundo a Lei 14.133/2021)
  • Critério 4 (alto valor sem competição): modalidade Dispensa ou Inexigibilidade com valor superior a R$ 10 milhões

Critérios excluídos e justificativa:

  • criterio_1_acima_p99_modalidade (1,01% positivos) — tautológico: define outlier estatístico, não regra de negócio
  • criterio_2_sem_competicao (63,49% positivos) — domina o sinal, tornaria classificação trivial

3. Feature engineering — substituindo valor_licitacao¶

valor_licitacao e modalidade_compra são usados internamente pelos critérios 3 e 4. Manter essas variáveis como features causaria o modelo aprender literalmente as regras (overfitting trivial).

Solução: criar a feature derivada razao_valor_mediana_modalidade — quanto a licitação vale em relação à mediana da modalidade. Captura "atipicidade contextual" sem reproduzir a regra exata.

In [15]:
# Feature derivada: razão valor / mediana da modalidade
df["mediana_modalidade"] = (
    df.groupby("modalidade_compra")["valor_licitacao"]
    .transform("median")
)
df["razao_valor_mediana"] = df["valor_licitacao"] / df["mediana_modalidade"]
df["log_razao_valor_mediana"] = np.log10(df["razao_valor_mediana"].clip(lower=0.01))

4. Definição do conjunto de features independentes¶

Excluídas (leakage):

  • valor_licitacao, log_valor_licitacao — usadas nos critérios 3 e 4
  • modalidade_compra — usada no critério 4
  • Todos os criterio_* — definem a label
  • anomalia_score, anomalia_label — saídas do Isolation Forest
  • candidato_anomalia — agregação dos critérios
  • Identificadores: numero_licitacao, codigo_ug, objeto, etc.

Mantidas (independentes):

  • Contagens: n_itens, n_participantes, valor_total_itens
  • Concentração: hhi, top1_share
  • Derivada: log_razao_valor_mediana
  • Temporais: dia_semana, mes
  • Categóricas: uf, situacao_licitacao

7. Treinamento dos três modelos¶

8. Avaliação comparativa¶

Métricas: ROC-AUC, Average Precision (PR-AUC), F1, Precision, Recall, Balanced Accuracy.

Por que essas métricas: com classe desbalanceada (~1%), accuracy é enganosa — um modelo que prevê tudo como "não-suspeita" teria 99% de accuracy e zero utilidade.

In [20]:
resultados = {}

    y_pred = pipe.predict(X_test)
    y_proba = pipe.predict_proba(X_test)[:, 1]

        "roc_auc": roc_auc_score(y_test, y_proba),
        "pr_auc": average_precision_score(y_test, y_proba),

Para contextualizar, aplicamos um Isolation Forest novo (não-supervisionado) sobre as mesmas features e comparamos com a label honesta.

# Isolation Forest aplicado nas mesmas features
preprocessor_if = ColumnTransformer(
    transformers=[
        ("num", StandardScaler(), FEATURES_NUMERICAS),
        ("cat", OneHotEncoder(handle_unknown="ignore", sparse_output=True), FEATURES_CATEGORICAS),
    ]
)

# IF não tem .fit_predict tradicional para sparse, então separamos
X_train_transformed = preprocessor_if.fit_transform(X_train)
X_test_transformed = preprocessor_if.transform(X_test)

if_model = IsolationForest(contamination=0.05, random_state=42, n_jobs=-1)
if_model.fit(X_train_transformed)

# Predição em teste (-1 = anomalia, 1 = normal); convertemos para 1=anomalia, 0=normal
y_if_pred = (if_model.predict(X_test_transformed) == -1).astype(int)

# Métricas do IF
print("Isolation Forest aplicado nas mesmas features:")
print(classification_report(y_test, y_if_pred, target_names=["normal", "suspeita"]))

resultados["Isolation Forest"] = {
